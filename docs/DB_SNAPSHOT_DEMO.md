## Sanitized Snapshot SQL

Run these read-only queries in Supabase SQL editor after seed:

```sql
select "nameRole" from "roles" order by "nameRole";

select "departmentName", "managerID"
from "departments"
order by "departmentName";

select u.email, u.username, r."nameRole", d."departmentName", u."isActive"
from "users" u
join "roles" r on r."roleID" = u."roleId"
left join "departments" d on d."departmentID" = u."departmentID"
order by u.email;

select "nameTypeLeave", "hasSalary"
from "type_leaves"
order by "nameTypeLeave";

select "month", "year", "status", "isSubmitted", "canSubmit", count(*) as count
from "monthly_timesheets"
group by "month", "year", "status", "isSubmitted", "canSubmit"
order by "year", "month", "status";

select "relatedType", "isRead", count(*) as count
from "notifications"
group by "relatedType", "isRead"
order by "relatedType", "isRead";
```

Do not export password hashes, refresh tokens, JWT secrets or Supabase credentials in demo evidence.
